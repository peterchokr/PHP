## 1️⃣ 세션의 개념과 작동 원리

### 1-1 세션이란?

**세션(Session)?** 서버에서 사용자 정보를 저장하는 방식입니다.

```
사용자가 로그인하면?
1. 서버가 사용자 정보를 저장 (메모리 또는 파일)
2. 사용자에게 세션 ID 할당
3. 사용자가 페이지를 방문할 때마다 세션 ID 확인
4. 서버가 저장된 정보 제공

특징:
- 서버에 저장됨 (안전)
- 세션 시간 설정 가능 (타임아웃)
- 로그아웃 시 삭제 가능
```

### 1-2 session_start() 함수

**목적:** 세션을 시작합니다. 모든 PHP 파일의 최상단에 작성합니다.

**함수 설명:**

```
session_start() 함수는:
- 서버에서 세션 데이터를 저장할 공간을 만들어줌
- 사용자마다 고유한 세션 ID를 생성
- 이 세션 ID를 쿠키로 클라이언트에 저장
- 이후 $_SESSION 배열을 사용할 수 있게 해줌
```

**코드:**

```php
<?php

// 세션 시작 (모든 PHP 파일의 맨 처음)
session_start();

// 이제 $_SESSION을 사용 가능
echo "세션이 시작되었습니다";

?>
```

**주의:** session_start()는 반드시 HTML 코드 출력 전에 실행해야 합니다.

```php
<?php
session_start();  // ✅ 올바른 위치
?>

<!DOCTYPE html>
<!-- HTML 코드 -->

<?php
// session_start();  // ❌ 여기는 너무 늦음
?>
```

### 1-3 $_SESSION에 데이터 저장

**목적:** 세션 배열에 사용자 정보를 저장합니다.

**배열이란?**

```
$_SESSION은 연관 배열(Key-Value 쌍)입니다.
Key(이름)을 통해 Value(값)를 저장하고 불러올 수 있습니다.

예시:
$_SESSION['user_id'] = 1;        // Key: 'user_id', Value: 1
$_SESSION['username'] = '홍길동'; // Key: 'username', Value: '홍길동'

이후 불러올 때:
echo $_SESSION['user_id'];      // 1 출력
echo $_SESSION['username'];     // 홍길동 출력
```

**코드:**

```php
<?php

session_start();

// 세션 데이터 저장
$_SESSION['user_id'] = 1;
$_SESSION['username'] = '홍길동';
$_SESSION['email'] = 'hong@example.com';

// 세션 데이터 사용
echo "사용자: " . $_SESSION['username'];

// 세션 데이터 확인
if (isset($_SESSION['user_id'])) {
    echo "로그인 상태";
}

?>
```

### 1-4 세션 종료

**목적:** 세션 데이터를 삭제합니다. 주로 로그아웃할 때 사용합니다.

**함수 설명:**

```
unset() - 특정 세션 데이터 삭제
  $_SESSION['user_id']를 삭제하면
  다시 접근할 때 isset($_SESSION['user_id'])는 false 반환

session_destroy() - 모든 세션 데이터 삭제
  서버에 저장된 모든 세션 정보를 삭제
  이 함수 이후로는 $_SESSION 사용 불가
  주로 로그아웃 시 사용
```

**코드:**

```php
<?php

session_start();

// 특정 세션 데이터 삭제
unset($_SESSION['user_id']);

// 모든 세션 데이터 삭제
session_destroy();

// 이제 $_SESSION 사용 불가

?>
```

## 2️⃣ 쿠키의 개념 (기초)

### 2-1 쿠키란?

**쿠키(Cookie)?** 클라이언트(브라우저)에 정보를 저장하는 방식입니다.

```
세션 vs 쿠키
┌─────────────┬──────────────────┬──────────────────┐
│ 항목        │ 세션             │ 쿠키             │
├─────────────┼──────────────────┼──────────────────┤
│ 저장 위치   │ 서버             │ 클라이언트(브라우저)
│ 보안        │ 높음             │ 낮음             │
│ 용도        │ 민감한 정보      │ 기억 정보        │
│ 용량        │ 제한 없음        │ 4KB 정도         │
│ 만료        │ 설정 가능        │ 지정된 시간      │
└─────────────┴──────────────────┴──────────────────┘
```

### 2-2 setcookie() 함수

**목적:** 브라우저에 쿠키를 저장합니다.

**함수 설명:**

```
setcookie(name, value, expire, path, domain);
- name: 쿠키 이름 (구분하기 위한 이름)
- value: 쿠키 값 (저장할 데이터)
- expire: 쿠키 만료 시간 (unixtime)
  time() + (86400 * 30) = 30일 후 삭제
- path: 쿠키가 유효한 디렉토리 (보통 '/')

주의: HTML 출력 전에 실행해야 함!
```

**코드:**

```php
<?php

// 쿠키 설정 (session_start() 전에 작성)
setcookie('user_theme', 'dark', time() + (86400 * 30)); // 30일 후 삭제
setcookie('language', 'ko');

// 쿠키 사용
echo $_COOKIE['user_theme'];

// 쿠키 삭제 (만료 시간을 과거로 설정)
setcookie('user_theme', '', time() - 3600);

?>
```

### 2-3 세션과 쿠키 사용 시점

```
세션을 사용할 때:
- 로그인 정보 저장
- 사용자 개인 정보 저장
- 보안이 중요할 때

쿠키를 사용할 때:
- 사용자 설정 저장 (테마, 언어)
- 자동 로그인 (옵션)
- 사용자 선호도 저장
```

## 3️⃣ 로그인/로그아웃 시스템

### 3-1 사용자 인증 프로세스

**로그인 과정:**

```
1. 사용자가 아이디, 비밀번호 입력
2. 서버가 데이터베이스에서 아이디 검색
3. 아이디 있으면 비밀번호 확인 (password_verify)
4. 비밀번호 일치하면 세션 저장
5. 로그인 완료
```

### 3-2 비밀번호 검증 (password_verify)

**목적:** 입력한 비밀번호가 저장된 해시와 일치하는지 확인합니다.

**함수 설명:**

```
password_verify($password, $hash);
- $password: 사용자가 입력한 평문 비밀번호
- $hash: 데이터베이스에 저장된 해시된 비밀번호

반환값:
- true: 비밀번호가 일치함
- false: 비밀번호가 일치하지 않음

내부 작동:
1. 입력한 비밀번호를 해시 함수로 변환
2. 변환된 해시와 저장된 해시를 비교
3. 같으면 true, 다르면 false 반환

중요:
- 저장된 해시를 볼 수 없으므로 안전함
- 원본 비밀번호를 절대 평문으로 저장하지 말 것!
```

**코드:**

```php
<?php

// 데이터베이스에 저장된 비밀번호 (해시)
$hashed_password = '$2y$10$eImiTXuWVxfaHNYY0iNAeuK2kRabMmwqbF/LewY5D...';

// 사용자가 입력한 비밀번호
$user_input = '12345678';

// password_verify()로 확인
if (password_verify($user_input, $hashed_password)) {
    echo "비밀번호 일치! 로그인 성공";
} else {
    echo "비밀번호 불일치";
}

?>
```

### 3-3 기본 로그인 로직

```php
<?php

session_start();

if ($_SERVER['REQUEST_METHOD'] === 'POST') {
  
    // 1단계: 입력값 확인
    if (empty($_POST['username']) || empty($_POST['password'])) {
        $error = "아이디와 비밀번호를 입력하세요";
    } else {
  
        // 2단계: 데이터베이스에서 사용자 검색
        $username = htmlspecialchars($_POST['username']);
  
        // (데이터베이스 연결 후)
        // Prepared Statement로 SQL Injection 방어
        $sql = "SELECT id, password FROM users WHERE username = ?";
        $stmt = $pdo->prepare($sql);
        $stmt->execute([$username]);
        $user = $stmt->fetch(PDO::FETCH_ASSOC);
        // $user = ['id' => 1, 'password' => '해시값...']
  
        // 3단계: 비밀번호 확인
        // password_verify(): 입력한 비밀번호가 저장된 해시와 일치하는지 확인
        // 반환값: true(일치) 또는 false(불일치)
        if ($user && password_verify($_POST['password'], $user['password'])) {
            // ✅ 비밀번호 일치: 로그인 성공
            $_SESSION['user_id'] = $user['id'];
            $_SESSION['username'] = $username;
            header("Location: mypage.php");
            exit;
        } else {
            // ❌ 비밀번호 불일치 또는 사용자 없음
            $error = "아이디 또는 비밀번호가 잘못되었습니다";
        }
    }
}

?>
```

### 3-4 로그아웃 기능

```php
<?php

session_start();

// session_destroy(): 모든 세션 데이터 삭제
// - 서버에 저장된 모든 $_SESSION 정보 제거
// - 사용자와 연결된 세션 ID도 무효화됨
// - 보안: 다른 사람이 세션을 탈취해도 더 이상 사용 불가
session_destroy();

// 로그인 페이지로 리다이렉트
// - 사용자가 로그인 화면으로 돌아감
// - 다시 아이디, 비밀번호를 입력해야 함
header("Location: login.php");
exit;

?>
```

## 4️⃣ 보안 고려사항

### 4-1 비밀번호 해싱

**왜 필요한가?**

```
❌ 위험한 방법: 비밀번호를 그대로 저장
- 데이터베이스가 해킹되면 모든 비밀번호 노출됨
- 사용자 정보가 완전히 탈취됨

✅ 안전한 방법: 비밀번호를 해시로 저장
- 데이터베이스가 해킹되어도 원본 비밀번호 알 수 없음
- 비밀번호 일치 여부만 확인 가능
```

**password_hash() 함수 설명:**

```
password_hash($password, $algo);
- $password: 사용자가 입력한 평문 비밀번호
- $algo: 해싱 알고리즘 (PASSWORD_DEFAULT 권장)

반환값:
- 암호화된 해시 문자열

예시:
입력: "12345678"
출력: "$2y$10$eImiTXuWVxfaHNYY0iNAeuK2kRabMmwqbF/LewY5D..."
(매번 다른 해시 생성됨 - 보안 때문)
```

**코드:**

```php
<?php

// 사용자가 입력한 비밀번호
$password = '12345678';

// 해시 생성 (데이터베이스에 저장)
$hashed = password_hash($password, PASSWORD_DEFAULT);

// 저장: INSERT INTO users (password) VALUES ('$hashed')

?>
```

### 4-2 로그인 확인

**페이지 접근 전 세션 확인:**

```php
<?php

session_start();

// isset($_SESSION['user_id']): 세션이 존재하는가?
// - true: 로그인된 사용자
// - false: 로그인하지 않은 사용자

// 로그인하지 않으면 로그인 페이지로 리다이렉트
if (!isset($_SESSION['user_id'])) {
    header("Location: login.php");
    exit;
    // exit;를 통해 이 아래 코드 실행 방지
}

// 여기 아래는 로그인한 사용자만 실행됨
echo "안녕하세요, " . $_SESSION['username'];

?>
```

## 5️⃣ 실습 예제

### 5-1 준비: 데이터베이스 및 users 테이블 생성

실습 예제를 실행하기 전에 먼저 users 테이블을 생성하고 샘플 데이터를 넣어야 합니다.

### 테이블 생성

MySQL Workbench 에서 다음 SQL을 실행하세요:

```sql
CREATE TABLE IF NOT EXISTS users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE,
    password VARCHAR(255) NOT NULL,
    email VARCHAR(100) NOT NULL,
    name VARCHAR(50) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_login TIMESTAMP NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;


```

### PHP에서 테스트 데이터 생성

테이블은 위 SQL로 만들고, 비밀번호만 PHP로 해싱하고 싶다면:

**파일명: `setup_users.php`** (1회만 실행)

```php
<?php

$host = 'localhost';
$dbname = 'test_db';
$user = 'root';
$password = '';

try {
    $pdo = new PDO(
        "mysql:host=$host;dbname=$dbname",
        $user,
        $password
    );
  
    // 사용자 정보 배열
    $users = array(
        array(
            'username' => 'hong',
            'password' => 'password123',
            'email' => 'hong@example.com',
            'name' => '홍길동'
        ),
        array(
            'username' => 'kim',
            'password' => 'kim12345',
            'email' => 'kim@example.com',
            'name' => '김영희'
        ),
        array(
            'username' => 'lee',
            'password' => 'lee@123456',
            'email' => 'lee@example.com',
            'name' => '이순신'
        )
    );
  
    // INSERT SQL 준비
    $sql = "INSERT INTO users (username, password, email, name) 
            VALUES (?, ?, ?, ?)";
    $stmt = $pdo->prepare($sql);
  
    // 각 사용자 삽입
    foreach ($users as $user) {
        // password_hash(): 비밀번호를 안전하게 해싱
        // PASSWORD_BCRYPT: bcrypt 알고리즘 사용 (권장)
        $hashed_password = password_hash($user['password'], PASSWORD_BCRYPT);
  
        $stmt->execute(array(
            $user['username'],
            $hashed_password,
            $user['email'],
            $user['name']
        ));
  
        echo "✅ " . $user['username'] . " 사용자 생성 완료<br>";
    }
  
    echo "<br>모든 사용자가 생성되었습니다!";
  
} catch (PDOException $e) {
    echo "❌ 오류: " . $e->getMessage();
}

?>
```

### 주요 포인트

✅ **테이블 구조:**

- `id`: 사용자 고유 ID (PRIMARY KEY, AUTO_INCREMENT)
- `username`: 로그인 아이디 (UNIQUE: 중복 불가)
- `password`: 해싱된 비밀번호 (255자: 해시값 길이)
- `email`: 이메일 주소
- `name`: 사용자 이름
- `created_at`: 가입 일시 (자동 생성)
- `last_login`: 마지막 로그인 시간

✅ **보안:**

- `password_hash()`: 비밀번호 안전하게 해싱
- `PASSWORD_BCRYPT`: bcrypt 알고리즘 (권장)
- `password_verify()`: 입력한 비밀번호와 해시 비교

✅ **로그인 테스트:**

```
아이디: hong
비밀번호: password123

아이디: kim
비밀번호: kim12345

아이디: lee
비밀번호: lee@123456
```

### 5-2 실습 예제: 기본 로그인 시스템

#### 파일 1: login.php (로그인 폼)

**코드:**

```php
<?php

session_start();

// 이미 로그인했으면 마이페이지로 이동
if (isset($_SESSION['user_id'])) {
    header("Location: mypage.php");
    exit;
}

$host = 'localhost';
$dbname = 'test_db';
$user = 'root';
$password = '';

try {
    $pdo = new PDO(
        "mysql:host=$host;dbname=$dbname",
        $user,
        $password
    );
} catch (PDOException $e) {
    die("연결 실패: " . $e->getMessage());
}

$error = '';

if ($_SERVER['REQUEST_METHOD'] === 'POST') {
  
    if (empty($_POST['username']) || empty($_POST['password'])) {
        $error = "아이디와 비밀번호를 입력하세요";
    } else {
  
        $username = htmlspecialchars($_POST['username']);
  
        $sql = "SELECT id, password FROM users WHERE username = ?";
        $stmt = $pdo->prepare($sql);
        $stmt->execute([$username]);
        $user_data = $stmt->fetch(PDO::FETCH_ASSOC);
  
        // password_verify(): 입력한 비밀번호와 해시 비교
        // $user_data: 데이터베이스에서 찾은 사용자 정보
        // password_verify(입력값, 저장된해시): true 또는 false 반환
        if ($user_data && password_verify($_POST['password'], $user_data['password'])) {
```

## 6️⃣ 주요 개념 정리

### 6-1 세션 함수 정리

```php
<?php

// 모든 PHP 파일의 맨 처음에 세션 시작
// - 서버에서 세션 저장소 초기화
// - $_SESSION 배열 사용 가능하게 함
session_start();

// 세션 데이터 저장 (연관 배열)
// - Key-Value 방식으로 데이터 저장
// - 다른 페이지에서도 $_SESSION으로 접근 가능
// - 사용자별로 독립적인 데이터 저장
$_SESSION['key'] = 'value';
$_SESSION['user_id'] = 1;
$_SESSION['username'] = '홍길동';

// 세션 데이터 확인
// - isset(): 변수가 존재하는가?
// - $_SESSION['key']가 존재하면 true, 없으면 false
if (isset($_SESSION['key'])) {
    echo $_SESSION['key'];
}

// 특정 세션 데이터만 삭제
// - unset()은 배열에서 특정 요소 제거
// - $_SESSION['user_id']만 삭제됨
// - $_SESSION['username']은 남아있음
unset($_SESSION['key']);

// 모든 세션 데이터 삭제
// - 서버에 저장된 모든 $_SESSION 정보 제거
// - 이 함수 이후로는 $_SESSION 사용 불가
// - 주로 로그아웃할 때 사용
session_destroy();

?>
```

### 6-2 비밀번호 처리

```php
<?php

// ===== 회원가입할 때 (비밀번호 저장) =====

// 사용자가 입력한 평문 비밀번호
$password = '12345678';

// password_hash(): 비밀번호를 안전하게 해시 변환
// - PASSWORD_DEFAULT: 현재 가장 안전한 알고리즘 사용
// - 반환값: 해시된 문자열 (매번 다른 결과)
// - 예: "$2y$10$eImiTXuWVxfaHNYY0iNAeuK2kRabMmwqbF/LewY5D..."
$hashed = password_hash($password, PASSWORD_DEFAULT);

// 데이터베이스에 저장 (평문이 아닌 해시를 저장!)
// INSERT INTO users (password) VALUES ('$hashed')

// ===== 로그인할 때 (비밀번호 확인) =====

// 사용자가 입력한 평문 비밀번호
$user_input = '12345678';

// 데이터베이스에서 조회한 해시
$hashed_from_db = $hashed;  // 데이터베이스의 해시 값

// password_verify(): 입력한 비밀번호가 저장된 해시와 일치하는가?
// - 첫 번째 인자: 사용자 입력 (평문)
// - 두 번째 인자: 데이터베이스의 해시
// - 반환값: true(일치) 또는 false(불일치)
if (password_verify($user_input, $hashed_from_db)) {
    // ✅ 비밀번호 일치: 로그인 성공
    echo "비밀번호 일치";
} else {
    // ❌ 비밀번호 불일치: 로그인 실패
    echo "비밀번호 불일치";
}

?>
```
